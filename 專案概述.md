# 專案概述

## 一、專案目的

本專案用來自動產生帕妃 PUFII 新品上架所需的多種 Excel 匯入檔。

原本新品上架流程需要人工從商品大檔、尺寸表、試穿報告、圖片資料、價格資料中複製欄位，再整理成不同平台需要的格式。這個專案把這些規則寫成 Python 程式，讓使用者只要放入固定格式的來源 Excel，就能一鍵產出：

1. 直播表格
2. 91 平台新品上架檔
3. 官網收音機匯入檔整包
4. 每個檔案的 audit 驗證紀錄
5. 統一待補資料清單
6. 流程與程式碼總檔

核心目標是：

- 減少人工複製貼上的錯誤。
- 固定欄位來源與輸出格式。
- 每次產出都留下可追溯紀錄。
- 把缺漏資料集中整理，方便一次補齊。
- 讓同一批資料可以反覆重跑，不靠人工記憶。

---

## 二、專案輸入資料

正式執行主要讀取 `輸入檔/` 內兩個 Excel：

| 輸入檔 | 功能 |
|---|---|
| `商品資料(大檔).xlsx` | 商品主資料來源，包含批次記號、用途、流水號、商品名稱、價格、成本、圖片、顏色尺寸、分類與官網匯入所需資料。 |
| `商品資料大檔.xlsx` | 上述大檔的 alias/symlink，用來相容部分程式舊檔名。 |
| `尺寸表&試穿報告.xlsx` | 尺寸表、推薦尺寸、試穿報告來源。直播表格與 91 上架檔會用流水號去對應。 |

篩選本批次商品的主要邏輯：

- `商品資料(大檔)` A 欄「記號」等於日期批次，例如 `0414`。
- B 欄「用途」等於 `上架`。
- E 欄「流水號」作為商品主鍵。

> 專案規則：商品辨識以流水號/物品編號為準，不用顏色認列商品。

## 三、專案提供的功能

### 1. 一鍵產生全部輸出

執行：

```bash
/opt/homebrew/bin/python3 程式/run_all_pafei_outputs.py --date 0414 --year 2026
```

這支總控程式會：

7. 產生 `帕妃自動產品產生檔-流程與程式碼總檔.md`，整理流程與程式碼。

執行順序如下：

| 9 | `generate_website_recommend_pending.py` | 官網推薦待填寫 |

### 2. 產生直播表格

程式：`程式/generate_live_table.py`

輸出：`輸出檔/0414直播表格.xlsx`

主要功能：

- 讀取商品大檔中本批次上架商品。
- 讀取尺寸表與試穿報告。
- 讀取 RAY 價格/成本資料。
- 整理成直播使用的多分頁表格。

目前表格分頁包含：

| 分頁 | 功能 |
|---|---|
| `{日期}` | 直播主表。 |
| `對款` | 對款資料。 |
| `姐` | 姐分頁資料。 |
| `RAY` | RAY 價格/成本相關資料。 |

運作方式：

1. 從商品大檔依日期與用途篩商品。
2. 以流水號到尺寸表/試穿報告找推薦尺寸與尺寸文字。
3. 以流水號/貨號對應價格、成本、新品價、直播價。
4. 將資料寫入四個分頁。
5. 產生 `.audit.json` 記錄輸入檔 hash、輸出檔 hash、缺漏欄位與待確認規則。

---

### 3. 產生 91 新品上架檔

程式：`程式/generate_91_listing.py`

輸出：`輸出檔/91-0414新品上架的檔案.xlsx`

主要功能：

- 產生 91 平台新品匯入格式。
- 固定輸出 `商品資料` 分頁。
- 依商品顏色/尺寸展開商品選項。
- 自動填入商品名稱、售價、成本、銷售日期、商品料號、商品特色、詳細說明、圖片等欄位。

運作方式：

1. 從商品大檔篩選 `用途=上架` 的商品。
2. 從顏色尺寸資料建立商品選項。
3. 每個有效的顏色/尺寸選項輸出成一列。
4. 商品料號依規則轉成 `CPxxxxx`。
5. 銷售開始日期由 `--date` 與 `--year` 組成。
6. 圖片路徑依商品資料與規則組合。
7. 缺顏色、尺寸、價格、成本等資料時不硬塞空白列，而是寫入 audit。

---

### 4. 用人工 91 檔回推驗證

程式：`程式/generate_91_from_manual_reference.py`

這支不是正式產檔主流程，而是開發/驗證用工具。

用途：

- 當商品大檔的日期/用途標記不完整，但人工 91 檔已有完整商品清單時，可以用人工 91 檔中的 `CPxxxxx` 商品料號回推 `Pxxxxx` 流水號。
- 再用商品大檔與尺寸表重新產測試版 91 檔。
- 最後比對人工檔與程式產物差異，找出規則是否需要修正。

運作方式：

1. 讀人工 91 檔。
2. 從商品名稱或商品料號抓出 `CPxxxxx`。
3. 轉回大檔流水號 `Pxxxxx`。
4. 到商品大檔抓商品資料。
5. 依 91 格式重建輸出列。
6. 比對商品名稱、價格、成本、銷售日期、商品選項、商品料號等重點欄位。
7. 輸出測試檔與差異報告。

### 5. 產生官網收音機匯入檔

官網收音機是一組多檔案匯入格式，本專案會在：

`輸出檔/0414官網上架匯入收音機的檔案/`

產出以下檔案。

#### 5.1 檔案2：名稱+大小

程式：`generate_website_radio.py`

輸出：`0414新品-檔案2-名稱+大小.xlsx`

功能：

- 產生商品基本資料。
- 包含物品編號、物品名稱、簡介、小圖、原價、價格、排序、重量、尺寸表連結、試穿報告連結、手機版原始碼、原價標等欄位。

運作方式：

1. 從商品大檔篩出本批次上架流水號。
2. 建立流水號對商品主資料的 map。
3. 建立流水號對價格資料的 map。
4. 組成官網檔案2欄位。
5. 寫出 Excel 與 audit。

#### 5.2 檔案3：色塊

程式：`generate_website_color_blocks_file3.py`

輸出：`0414新品-檔案3-色塊.xlsx`

功能：

- 產生每個商品顏色對應的色塊與圖片路徑。

運作方式：

1. 找出本批次上架流水號。
2. 從大師分頁讀取商店顏色、圖片路徑、色塊號碼。
3. 用「同一流水號 + 商店顏色」去重，避免尺寸列重複造成色塊重複。
4. 組成色塊路徑、小圖、大圖、超大圖、排序、銷售欄位。
5. 寫出 Excel 與 audit。

#### 5.3 檔案5：中圖合圖

程式：`generate_website_middle_images_file5.py`

輸出：`0414新品-檔案5-中圖合圖.xlsx`

功能：

- 產生商品明細頁多小圖/中圖/大圖路徑。

運作方式：

1. 找出本批次上架流水號。
2. 從大師分頁讀取圖片路徑與中圖張數。
3. 以「流水號 + 顏色」去重。
4. 依中圖張數展開 `_01` 到 `_NN` 的圖片列。
5. 寫出 Excel 與 audit。

#### 5.4 檔案4：隱藏分類

程式：`generate_website_hidden_category_file4.py`

輸出：`0414新品-檔案4-隱藏分類-1給客服檢查.xlsx`

功能：

- 將本批次每個上架商品放入固定隱藏分類。

運作方式：

1. 從商品大檔篩出上架商品。
2. 每個流水號輸出一列。
3. 分類編號固定為 `1370`。
4. 寫出 Excel 與 audit。

#### 5.5 檔案4：上架分類

程式：`generate_website_category_file4.py`

輸出：`0414新品-檔案4-上架分類-2可以上架.xlsx`

功能：

- 將本批次每個商品放入正式上架分類。

運作方式：

1. 從商品大檔篩出上架商品。
2. 每個商品固定展開兩列。
3. 分類編號分別為 `1701`、`1535`。
4. 寫出 Excel 與 audit。

#### 5.6 檔案7：洗滌+通用+標籤

程式：`generate_website_tags_file7.py`

輸出：`0414新品-檔案7-洗滌+通用+標籤-2.xlsx`

功能：

- 為每個商品產生洗滌說明、商品標籤、列表優惠 icon 等 HTML 內容。

運作方式：

1. 從商品大檔篩出上架商品流水號。
2. 每個商品固定產生三列：
   - `ext03`：洗滌說明，內容來自 `程式/templates/file7_ext03_washing.html`。
   - `label`：限時 SALE 標籤 HTML。
   - `listcustomicon`：優惠 icon 與活動連結 HTML。
3. 寫出 Excel 與 audit。

#### 5.7 推薦待填寫

程式：`generate_website_recommend_pending.py`

輸出：`0414新品-推薦(待填寫).xlsx`

功能：

- 產生推薦商品關聯的空白模板。

運作方式：

1. 從商品大檔篩出上架商品。
2. `主物品編號` 填入商品流水號。
3. `推薦物品編號` 與 `排序` 先留空。
4. 待人工後續填寫。
5. 寫出 Excel 與 audit。

## 四、Audit 與可追溯機制

本專案每次產生 Excel 時，都會同步產生同名 `.audit.json`。

每份 audit 通常記錄：

| 欄位 | 說明 |
|---|---|
| `generated_at` | 產生時間。 |
| `script` | 產出該檔案的程式路徑。 |
| `date_code` | 執行日期批次，例如 `0414`。 |
| `input` / `inputs` | 使用到的輸入檔。 |
| `input_sha256` | 輸入檔 hash。 |
| `output` | 輸出檔路徑。 |
| `output_sha256` | 輸出檔 hash。 |
| `row_count` | 輸出資料筆數。 |
| `missing` / `pending` / `missing_or_pending` | 缺漏資料或待確認規則。 |

用途：

- 證明檔案是由哪支程式產生。
- 確認當時使用哪一批輸入資料。
- 檢查輸出是否被人工改過。
- 讓缺漏欄位可以被集中整理。

## 五、待補資料彙整機制

總控程式 `run_all_pafei_outputs.py` 會掃描所有 `.audit.json`，把各檔案的缺漏項目統一整理成：

`輸出檔/待補資料.txt`

整理邏輯：

1. 從 `missing_or_pending`、`missing`、`pending` 抓資料。
2. 將不同格式的缺漏紀錄標準化。
3. 用流水號/物品編號辨識商品。
4. 依欄位名稱推估要回補的來源，例如：
   - 成本 → 商品資料大檔 K 欄 ray 成本。
   - 尺寸/推薦 → 尺寸表與試穿報告。
   - 顏色/色塊 → 商品資料大檔顏色尺寸資料。
   - 價格/直播價 → 商品資料大檔價格欄位。
5. 去除重複項目。
6. 分成「依流水號整理」與「依原始來源彙總」兩個角度輸出。

這讓客戶或維護者可以一次知道：

- 哪些流水號缺資料。
- 缺哪個欄位。
- 要去哪個原始檔補。
- 會影響哪個輸出檔。

## 七、專案目前已知規則

| 規則 | 說明 |
|---|---|
| 商品主鍵 | 以流水號/物品編號為主，不用顏色認列商品。 |

| 官網分類 | 隱藏分類固定 1370；上架分類固定 1701、1535。 |
| 檔案7 | 每個商品固定展開 `ext03`、`label`、`listcustomicon` 三列。 |

## 九、專案價值

此專案把帕妃新品上架流程從人工 Excel 整理，轉成可重跑、可稽核、可維護的自動化流程。

對業務端的價值：

- 上架檔產生更快。
- 欄位來源更清楚。
- 缺資料集中回補，不會散落在多個檔案。
- 每次輸出都有 audit，可追溯。

對維護端的價值：

- 每種輸出格式都有獨立程式，方便單獨修規則。
- 總控程式負責流程整合，避免漏跑某個檔案。
- 開發筆記與規格文件保留在專案內，方便後續接手。
- `generate_91_from_manual_reference.py` 可用人工正確檔反推驗證，降低規則重建風險。
