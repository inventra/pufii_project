# 專案的程式碼結構

本專案是「帕妃 PUFII 商品上架檔自動產生工具」，主要用 Python 讀取客戶提供的 Excel 原始資料，批次產生直播表格、91 上架檔、官網收音機匯入檔，以及每個輸出檔的 audit 檢查紀錄。

## 根目錄

| 路徑 | 類型 | 功能 |
|---|---|---|
| `程式/` | Python 程式碼 | 所有自動產檔邏輯、模板、執行說明。 |
| `輸入檔/` | Excel 輸入資料 | 正式執行時讀取的來源檔，包含商品大檔與尺寸/試穿報告。 |
| `輸出檔/` | 產出結果 | 程式產生的 Excel、audit JSON、待補資料與流程總檔。 |
| `流程筆記/` | Markdown 規格筆記 | 開發與比對過程中的欄位規則、格式差異、待確認問題。 |
| `Drive下載/` | 客戶原始交付檔 | 從雲端下載的原始參考檔/人工正確檔，供比對與回推規則使用。 |
| `.gitignore` | Git 設定 | 排除虛擬環境、快取、macOS 系統檔等不應上傳內容。 |
| `專案的程式碼結構.md` | 文件 | 說明各資料夾與程式檔負責功能。 |
| `專案概述.md` | 文件 | 說明專案目的、功能、運作流程與使用方式。 |

## `程式/`：核心程式碼

### `程式/run_all_pafei_outputs.py`

一鍵總控程式，負責重跑整包輸出。

| 區段/函式 | 具體功能 |
|---|---|
| `SCRIPTS` | 定義整包流程執行順序：直播表格 → 91 上架 → 官網檔案 2/3/5/4/7/推薦。 |
| `SCRIPT_FLOW_NAMES` | 把程式檔名轉成業務流程名稱，用於總檔報表。 |
| `norm()` | 統一把空值、整數型 float、字串轉成乾淨文字。 |
| `sha256()` | 計算檔案 SHA256，供 audit 驗證輸入/輸出是否一致。 |
| `ensure_big_file_alias()` | 確保 `輸入檔/商品資料大檔.xlsx` alias 存在；若只有 `商品資料(大檔).xlsx` 則建立 symlink。 |
| `run_generators()` | 依序呼叫每支產檔程式；任一支失敗就中止並報錯。 |
| `item_code()` | 從 audit 缺漏資料中抓出流水號/物品編號，作為商品辨識主鍵。 |
| `normalize_missing_item()` | 把各支程式 audit 裡格式不同的缺漏欄位統一整理成 `code / field / reason / source / output`。 |
| `infer_source()` | 依缺漏欄位名稱推估應該回補哪個來源檔欄位，例如成本、尺寸、顏色、價格。 |
| `collect_audit_missing()` | 掃描 `輸出檔/**/*.audit.json`，彙整所有缺漏與待補項目。 |
| `dedupe_missing()` | 移除重複缺漏紀錄。 |
| `write_missing_report()` | 產生 `輸出檔/待補資料.txt`，依流水號與來源整理要補的欄位。 |
| `summarize_outputs()` | 統計輸出 Excel 數、audit 數、待補資料 hash。 |
| `audit_by_script()` | 依 script 檔名索引各輸出 audit，供流程總檔使用。 |
| `write_workflow_code_bundle()` | 產生 `帕妃自動產品產生檔-流程與程式碼總檔.md`，包含輸出總表與每支程式碼內容。 |
| `main()` | 解析 CLI 參數 `--base --date --year --keep-output`，清空/建立輸出資料夾、執行產檔、寫總 audit。 |

### `程式/generate_live_table.py`

產生 `{日期}直播表格.xlsx`，供直播/客服使用。

| 區段/函式 | 具體功能 |
|---|---|
| `MAIN_HEADERS / MATCH_HEADERS / SISTER_HEADERS / RAY_HEADERS` | 定義不同分頁欄位：主表、對款、姐、RAY。 |
| `PENDING_RULES` | 記錄尚待客戶確認、程式暫不自動定案的規則。 |
| `norm()` | 清理字串與空值。 |
| `header_key()` | 比對 Excel 欄名時去掉空白與換行，避免欄名格式不同導致找不到欄位。 |
| `sha256()` | 產生輸入/輸出檔驗證 hash。 |
| `mmdd_today()` | 預設日期參數。 |
| `find_header()` | 在 Excel 中尋找指定欄名位置。 |
| `lookup_big_rows()` | 從商品大檔抓出指定日期/用途的商品資料。 |
| `lookup_ray_prices()` | 從 ray 價錢相關分頁建立價格、成本、新品價查詢。 |
| `lookup_color_map()` | 建立流水號對顏色資料的對照。 |
| `extract_variant_size()` | 從款式/尺寸文字中解析尺寸資訊。 |
| `normalize_body_size_text()` | 整理尺寸文字格式。 |
| `size_tokens()` | 把尺寸字串拆成可比對 token。 |
| `format_recommended_fit()` | 組合推薦尺寸文字。 |
| `build_size_maps()` | 從 `尺寸表&試穿報告.xlsx` 各尺寸分頁建立尺寸表/試穿報告對照。 |
| `value_at()` | 安全讀取工作表指定儲存格。 |
| `append_field()` | 組合尺寸欄位文字，保留來源欄名。 |
| `build_size_text_for_row()` | 依商品類型/流水號產生尺寸表文字。 |
| `write_headers()` | 寫入 Excel 表頭並套用樣式。 |
| `autosize()` | 自動調整欄寬。 |
| `write_missing_report()` | 寫出直播表格缺漏資料提示。 |
| `generate()` | 主產檔流程：讀輸入、組四個分頁、輸出 Excel、輸出 audit。 |
| `main()` | CLI 入口，接收 `--base --date --dry-run`。 |

### `程式/generate_91_listing.py`

產生 `91-{日期}新品上架的檔案.xlsx`，供 91 平台新品上架匯入。

| 區段/函式 | 具體功能 |

| `build_fit_map()` | 從尺寸/試穿報告建立推薦尺寸、尺寸表等對照。 |
| `build_option_map()` | 依 `顏色尺寸請用這一個` 展開商品選項；一個顏色尺寸為一列。 |
| `format_jpg_path()` | 依規則組圖片檔路徑。 |
| `build_image_path_map()` | 從大師/圖片資料建立商品圖片路徑對照。 |
| `load_listing_rows()` | 從商品大檔篩選用途為上架的商品，並組成 91 欄位。 |
| `autosize()` | 自動調整欄寬。 |
| `generate()` | 產生 Excel 與 audit，缺資料不輸出空白選項列，改寫入 audit。 |
| `main()` | CLI 入口，接收 `--base --date --year --dry-run`。 |

| `parse_manual_91()` | 讀人工 91 檔，從商品料號 `CPxxxxx` 回推流水號 `Pxxxxx`，收集選項。 |
| `build_big_map()` | 從商品大檔建立流水號對商品資料的 map。 |
| `build_fit_map()` | 建立尺寸/試穿資料 map。 |
| `make_row()` | 用大檔資料與人工選項組成一列 91 輸出資料。 |
| `write_workbook()` | 寫出測試版 Excel。 |
| `compare_rows()` | 比對人工檔與程式產物欄位差異。 |
| `generate()` | 執行回推、產測試檔、產差異報告。 |
| `main()` | CLI 入口。 |
