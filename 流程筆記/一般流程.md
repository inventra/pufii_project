## 尺寸表修正：兩件式需用 `//` 分隔

### 問題原因
- `上衣+小可愛(兩件式)` 分頁有兩組尺寸欄位。
- 之前因為分頁內有重複欄名，例如兩個 `全長`，程式只用欄名查找時會抓到後面那組，導致 P33315 組成：`肩寬39 袖長13 袖寬17 胸寬40 全長49`。
- 正確做法是兩組尺寸分開組合，中間用 `//`。

### P33315 正確尺寸表
來源：`尺寸表&試穿報告.xlsx` → `上衣+小可愛(兩件式)` 第 37 列。

輸出：
`肩寬39 袖長13 袖寬17 胸寬40 全長54 // 袖寬26 胸寬45 全長49`

### 規則
- 第一組：上衣尺寸。
- 第二組：小可愛/下身/第二件尺寸。
- 兩組中間使用 ` // ` 區隔。
- 空白欄位不輸出。


## 顏色欄位規則補充

### 來源分頁
顏色不是直接從尺寸表抓，而是由員工先在大檔內填入：

1. `本次要匯入的流水號`
   - A 欄：`要匯入的流水號`
   - B 欄：`顏色選項個數`

2. `顏色尺寸請用這一個`
   - A 欄：`物品編號`
   - H 欄：`顏色`

### 顏色產生規則
以每個直播商品流水號為 key：

1. 先到 `本次要匯入的流水號` 找該流水號。
2. 取得該流水號的 `顏色選項個數`，例如 3。
3. 再到 `顏色尺寸請用這一個` 找相同 `物品編號` 的所有列。
4. 做第二層防護：
   - `顏色尺寸請用這一個` 中該流水號的列數，必須等於 `本次要匯入的流水號` 的 `顏色選項個數`。
   - 若數量不一致，應標記異常，不要安靜帶錯。
5. 若數量一致，取出該流水號所有列的 H 欄 `顏色`。
6. 用 `、` 串接後填入直播表 `顏色` 欄。

### 範例
如果：
- `本次要匯入的流水號`：P33344 顏色選項個數 = 3
- `顏色尺寸請用這一個`：P33344 有 3 列，H 欄為 `白F`、`深灰F`、`黑F`

則直播表 `顏色` 輸出：
`白F、深灰F、黑F`

### 注意
- 目前 0414 測試資料中的 P33034、P33315 在 `本次要匯入的流水號` 與 `顏色尺寸請用這一個` 找不到，代表這份大檔目前的顏色分頁資料不是 0414 那批，或尚未由員工填入。
- 未來正式流程需先確認員工已完成 `本次要匯入的流水號` 填寫，才可產生顏色。


## 顏色欄位目前決議與格式規則

### 目前決議
- 直播表 `顏色` 欄目前先空白，不直接從 `流水號` 分頁 H 欄補。
- 顏色來源差異已列入待確認問題包。

### 未來若抓到顏色，輸出格式
- 多個顏色之間使用 `、` 分隔。
- 如果來源是 `{顏色}{尺寸}` 或 `{顏色}-{尺寸}` 形式，需要先依顏色整合尺寸。
- `F` 代表 free size，不特別再標一次尺寸。

### 整合範例
來源：
```text
淺藍S
淺藍M
藍S
藍M
藍L
藍XL
```

輸出應整合為：
```text
淺藍S-M、藍S-XL
```

如果尺寸中間跳號，例如只有 S、L：
```text
黑S/L
```

如果是 F：
```text
白
```
不要輸出成 `白F`。
