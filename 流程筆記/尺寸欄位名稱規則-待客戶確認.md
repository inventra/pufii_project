# 尺寸欄位名稱規則 - 待客戶確認

## 使用者補充
尺寸表中可能同時出現不同但相近的欄位名稱，例如：

- `下擺`
- `下擺寬`
- `大腿`
- `大腿寬`

這些不能由程式自行統一改名，因為客戶可能有固定用語。

## 目前暫定規則
輸出尺寸文字時，原則上保留來源欄位名稱：

| 來源欄位 | 輸出文字 |
|---|---|
| 下擺 | 下擺 |
| 下擺寬 | 下擺寬 |
| 大腿 | 大腿 |
| 大腿寬 | 大腿寬 |

也就是：來源寫什麼，輸出就寫什麼。

## 要問客戶
請確認客戶直播表尺寸文字希望使用哪一種命名：

1. 完全照尺寸表欄位名稱輸出？
   - 例如來源是 `大腿` 就輸出 `大腿28`
   - 來源是 `大腿寬` 就輸出 `大腿寬28`

2. 還是客戶希望統一成特定名稱？
   - 例如不管來源是 `大腿` 或 `大腿寬`，都輸出其中一種
   - 例如不管來源是 `下擺` 或 `下擺寬`，都輸出其中一種

3. 若同一分頁不同商品欄位名稱不一致，以哪個為準？

## 目前已知例子

### P33115 / 上衣+下身
來源欄位 Q-W：

| 欄位 | 來源名稱 | 值 |
|---|---|---|
| Q | 腰寬 鬆緊帶x~x | 30-41 |
| R | 臀寬 | 46 |
| S | 大腿 | 28 |
| T | 褲檔 | 33 |
| U | 下擺 | 空白 |
| V | 全長 | 34 |
| W | 褲管 | 空白 |

依目前暫定規則輸出：

`腰寬30-41 臀寬46 大腿28 褲檔33 全長34`

不可自行改成：

`大腿寬28`

除非客戶確認要統一成 `大腿寬`。
