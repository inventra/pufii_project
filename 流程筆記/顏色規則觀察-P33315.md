# 顏色規則觀察｜P33315

## 使用者實測
使用者在 `本次要匯入的流水號` 輸入 `P33315` 並設定顏色選項個數後，`顏色尺寸請用這一個` 會在下方產生對應列。

截圖看到 P33315 產生 4 列，H 欄顏色為：

```text
白F、粉F、黑F、白F
```

## 與流水號備註不一致
但 `商品資料大檔.xlsx` → `流水號` 分頁中，P33315 的備註欄是：

```text
米、黃、粉、灰
```

因此目前確認：

- `流水號` 分頁備註欄不一定等於正式顏色輸出。
- `顏色尺寸請用這一個` 產出的 H 欄可能來自員工後續選擇/調整，不是單純從流水號備註直接拆出。
- 自動化時不能直接用 `流水號` 備註欄當顏色答案。

## 目前規則暫定
顏色欄仍以雙層防護為主：

1. `本次要匯入的流水號` 找該流水號與顏色選項個數。
2. `顏色尺寸請用這一個` 找相同物品編號。
3. 確認找到列數等於顏色選項個數。
4. 取 H 欄 `顏色`，用 `、` 串接。

## 待觀察
如果 H 欄結果與流水號備註不一致，需確認：

- 員工是否會人工調整 `顏色尺寸請用這一個` 的 J/K 欄。
- H 欄公式是否依 J 欄商店顏色 + K 欄商店尺寸組成。
- 是否應完全信任 H 欄，而不是流水號備註。
